# CP-CTM 启动指南

## 环境要求

- **Node.js** >= 18.17
- **pnpm**（推荐）或 npm / yarn

```bash
# 检查 Node.js 版本
node -v

# 安装 pnpm（如未安装）
npm install -g pnpm
```

---

## 1. 安装依赖

```bash
# 进入项目目录
cd D:\REAL\CS\claude code\Cross-Platform Calendar Task Manager

# 安装所有依赖
pnpm install
```

## 2. 环境变量配置

项目根目录下必须存在 `.env.local` 文件，包含 Supabase 配置：

```
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6Ik...
```

> `.env.local` 已存在于项目中，无需重复创建。如丢失，从 Supabase Dashboard > Settings > API 中复制。

---

## 3. 启动方式

### 方式 A：生产模式（推荐，更稳定）

```bash
# 构建 + 启动（二合一）
pnpm build && pnpm start
```

- 默认地址：**http://localhost:3000**
- 端口被占用时自动递增（3001, 3002...）
- CSS 加载稳定，适合功能验证

### 方式 B：开发模式（带热更新）

```bash
pnpm dev
```

- 默认地址：**http://localhost:3000**
- 修改代码后自动刷新
- 偶发 CSS 加载异常，刷新页面即可恢复

---

## 4. 确认启动成功

终端出现以下信息表示启动完成：

```
▲ Next.js 14.x.x
- Local:        http://localhost:3xxx
```

用浏览器打开显示的地址即可访问。

## 5. 功能检验清单

| 功能 | 检验方法 |
|------|----------|
| **首页** | 访问 `/`，看到 CP-CTM 标题 + Sign in / Get started 按钮 |
| **注册** | 点击 Get started → 填写邮箱密码 → 注册成功跳转 Dashboard |
| **登录** | 点击 Sign in → 输入已注册的邮箱密码 → 进入 Dashboard |
| **日历视图** | 月视图/周视图切换，左右箭头翻月 |
| **农历信息** | 日期格内显示农历日期、节气名称和节假日名称 |
| **创建任务** | 点击日历标题旁 **+** 按钮，或日期格内的 **+ Add** → 弹窗填写标题 → 保存 |
| **编辑任务** | 右侧面板点击任务卡片 → 修改标题/描述/优先级 → 保存 |
| **状态切换** | 点击任务卡片上的状态徽章 → `todo → in-progress → done` 循环切换 |
| **删除任务** | 点击任务卡片上的删除图标 → 确认 → 任务消失 |
| **实时同步** | 打开两个浏览器窗口登录同一账号 → 任一窗口增删改任务 → 另一窗口自动更新 |
| **登出** | 侧边栏底部 Logout → 回到首页 |

## 6. 常见问题

### 端口被占用

```bash
# 查看占用 3000 端口的进程
netstat -ano | findstr :3000

# 杀掉进程（PID 替换为实际值）
taskkill -F -PID 12345
```

### 构建失败

```bash
# 清理缓存后重试
rm -rf .next
pnpm build
```

### 开发模式下 CSS 不生效

访问 `http://localhost:3xxx/_next/static/css/app/layout.css` 如果返回 HTML 而非 CSS 内容：
- 刷新页面（F5）通常可解决
- 或切换到生产模式（`pnpm build && pnpm start`）
